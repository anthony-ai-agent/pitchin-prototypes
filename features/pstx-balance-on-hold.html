PSTX Account — “ECF investment” line in Balance on Hold

01

What’s changing

Today the Balance-on-Hold card shows a headline held amount plus two drillable rows — PSTX Open Orders and Withdrawal requests. This adds a third row, ECF investment, that behaves identically. Its subtotal is the sum of the investor’s ECF pledges in Paid status. Expanding it lists each pledge and links to the ECF Investments tab, pre-filtered to Paid.

  • New row copy “ECF investment”, tagged NEW (remove tag after launch), positioned third.
  • Same drill-down as existing rows: desktop → anchored popover, mobile → bottom sheet.
  • A zero-value bucket is inert (no chevron, not clickable) — identical to the shipped rows.
  • The rest of the screen (top nav, page title + PSTX badge, Account balance card with Top Up / Withdraw) is unchanged — shown for context.

Edge states

One edge state to handle: the investor has no ECF pledges on hold.

No ECF investment (amount 0) → greyed, disabled, no popover

No ECF investment → disable the row. When ecfInvestmentHoldAmount is 0 (investor has no Paid pledges on hold), the row is greyed out and not clickable: render the <button> as [disabled] — label + amount greyed (#B8BCC4), no chevron, no hover, and no popover / bottom sheet appears. Identical to how PSTX Open Orders and Withdrawal requests already render at 0.

07

Interaction & responsive

  • Drillable only when non-zero. If ECF hold = 0: no chevron, not focusable/clickable, no hover.
  • One open at a time. Opening a bucket closes any other open bucket.
  • Re-click toggles closed (no re-fetch). Records may be fetched on first open and cached.
  • Dismiss: outside click, ×, or Esc. Mobile: backdrop tap / swipe-down.
  • Breakpoint: window.innerWidth < 768 → bottom sheet; else → anchored popover.
  • Placement: bottom-start, container body, auto-flip up on overflow.
  • List overflow: record list caps max-height 260px and scrolls internally.
  • A11y: row is a real <button> with aria-expanded; visible focus ring; × has aria-label.
08

Data contract & navigation

The UI binds to two new pieces on the investor wallet-details response (backend slice — AC1). Shapes are the contract; confirm names with the API team.

// Extend the investor wallet-details response
interface WalletDetails {
  balance: number;
  pendingAmount: number;                // headline "Balance on Hold"
  openOrdersHoldAmount: number;         // existing — PSTX Open Orders
  withdrawalRequestsHoldAmount: number; // existing — Withdrawal requests
  ecfInvestmentHoldAmount: number;      // NEW — ECF investment subtotal
}

// Drill-down records (fetched on first open, like heldOpenOrders)
interface HeldEcfInvestment {
  campaignId: number;      // optional deep link to the campaign
  companyName: string;     // → record primary
  paidAt: string;          // ISO date → "Paid · {{date}}"
  amount: number;          // → record amount
}

“View all in ECF Investments” target — resolved from the repo:

router.navigate(['/dashboard/pledges'], { queryParams: { payment: 'Paid' } });
// EcfInvestmentsPreviewComponent reads queryParams['payment'];
// accepts the enum NAME or number. InvestmentStatus.Paid = 2 → ?payment=Paid | 2
Reconciliation source of truth: sum of drill-down record amounts == ecfInvestmentHoldAmount, and the three subtotals == pendingAmount. Verify against real data, not mocks.
